================================================================================
OVERVIEW
================================================================================

Goal: display a team crest/logo image in the Team column of rendered matplotlib
tables, alongside (or replacing) the text abbreviation.

matplotlib.table cells are text-only. Images must be placed as a post-render
overlay using OffsetImage + AnnotationBbox, positioned to align with each
cell's bounding box after the table is drawn.

================================================================================
RECOMMENDED APPROACH — OffsetImage + AnnotationBbox (local assets)
================================================================================

1. Maintain a local logo directory:
     assets/logos/{team_abbr}.png
   e.g. assets/logos/LOU.png, assets/logos/CHS.png, etc.

2. Add optional parameters to render_table():
     logo_col:  str | None = None     # which column holds team abbreviations
     logo_dir:  str | None = None     # path to the logos directory

   When both are provided, the post-render logo pass runs.
   When either is None, behavior is identical to today — fully backward-compatible.

3. Post-render logo placement logic (runs after tbl is drawn and fig is
   rendered to canvas):

     a. Call fig.canvas.draw() to force layout computation — this is required
        before cell bounding boxes are available.

     b. Identify the column index of logo_col in df.columns.

     c. Iterate over data rows (tbl_row = 1..n):
          - Retrieve cell: cell = tbl[tbl_row, col_idx]
          - Get bounding box in display coords: bbox = cell.get_window_extent()
          - Convert to axes coords using ax.transData.inverted() or
            ax.transAxes — AnnotationBbox works in axes or figure coords.
          - Load logo image: img = plt.imread(logo_dir / f"{abbr}.png")
          - Wrap: imagebox = OffsetImage(img, zoom=<scale>)
          - Place: ab = AnnotationBbox(imagebox, xy=cell_center,
                       xycoords="figure pixels", frameon=False)
            ax.add_artist(ab)

     d. Optionally clear the text in the Team cell (cell.get_text().set_text(""))
        if the logo fully replaces the name, or offset the logo left and keep
        abbreviated text to the right.

4. Zoom/scale: cell height in points ≈ (figsize_inches * 72) / (n_rows + 1).
   A zoom factor of roughly cell_height_pts / (logo_native_height_px * 72/dpi)
   keeps the logo within the cell. In practice, start with zoom=0.035–0.06
   and adjust by table size.

================================================================================
LOGO SOURCE OPTIONS
================================================================================

Option A — Local assets/ directory (RECOMMENDED)
  Path:    assets/logos/{team_abbr}.png
  Pros:    No network dependency, consistent quality, works offline,
           full control over sizing and format.
  Cons:    Manual maintenance — must add new teams each season, update
           rebrandings (e.g. Hartford to Rhode Island).
  Format:  PNG with transparency (RGBA), square crop, ~256x256px minimum.
  Source:  Wikipedia SVG exports (convert to PNG), club official media kits,
           or Wikimedia Commons (verify license per club).

Option B — URL fetch at render time
  Pros:    No local files to maintain.
  Cons:    Network latency on every render, URLs break without warning,
           rate limiting, caching needed, licensing unclear for ESPN/Wikipedia.
  Not recommended for a stable publishing workflow.

Option C — plottable library (significant refactor)
  pip install plottable
  plottable.Table supports ColDef(plot_fn=...) for image columns natively.
  Pros:    Cleaner cell-level image support, better layout control.
  Cons:    Requires rewriting render_table() substantially; current
           accent_color, rank_cols, col_formats logic would all need porting.
  Worth evaluating if render_table() ever needs a broader overhaul.

================================================================================
ITEMS FOR CONSIDERATION
================================================================================

Coverage
  - USLC roster changes each season (promotion, relegation, expansion).
    New clubs need logos before the season begins.
  - Track which abbreviations map to which logo files in a manifest or simply
    rely on the filename convention {team_abbr}.png.
  - Multi-team player rows (e.g. "LOU/IND") have no single logo — skip or
    show both at reduced size.

Fallback behavior
  - If a logo file is missing for a team, fall back to the text abbreviation
    silently (no error). Log a warning at most.
  - If logo_dir does not exist, skip the logo pass entirely.

Cell text
  - Decide per-table: replace abbreviation with logo only, or show logo +
    full team name. Full name requires wider Team column (wider figsize).
  - For the standings table the abbreviation is already readable — logo-only
    or logo+abbr both work.

Sizing consistency
  - All logos in a column must render at the same height. Derive zoom from
    actual cell height (from fig.canvas.draw() bounding boxes) rather than
    hardcoding, so it scales with figsize and row count.

DPI and export
  - fig.savefig(..., dpi=150) at higher resolution will shift pixel bounding
    boxes. Test logo placement at the same DPI used for final export.
  - AnnotationBbox in figure-pixel coords is DPI-sensitive; axes-fraction
    coords (xycoords="axes fraction") are more portable.

render_table() signature change
  - New parameters should be keyword-only and optional — no breaking change
    to existing callers.
  - Suggested addition:
      logo_col: str | None = None,
      logo_dir: str | os.PathLike | None = None,

Assets directory structure (suggested)
  ASA_SOCCER_DATA/
    assets/
      logos/
        uslc/
          LOU.png
          CHS.png
          SAC.png
          ... (one file per team abbreviation)
        mls/        # if MLS tables are ever added
        usl1/
